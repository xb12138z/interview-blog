## 一维数组

### 两数之和

思路：

使用哈希表。

代码：

```cpp
vector<int> twoSum(vector<int>& nums, int target) {
    unordered_map<int,int> mp;

    for(int i=0;i<nums.size();i++){
        int t = target - nums[i];
        if(mp.count(t))
            return {mp[t],i};
        mp[nums[i]] = i;
    }

    return {};
}
```

## 子数组

面对子数组方面的问题，我们有时候不要陷入思考数学解或者思考遍历方面的方法。不妨试试逆向思维  - -   想子数组以本位作为结尾的情况，如下方 2348全0子数组的数目、 1504统计全1子矩阵。

### 1.2348全0子数组的数目 ： 

思路：

在计算形如 00..0 这种数组的全0子数组时我们可以直接cur++，res+=cur。因为每次子数组多一个0时，已这个0结尾构成的额外子数组数量 =  前面所有0的个数的总和。


### 2.1504统计全1子矩阵：

思路：

方法一：想办法将二维的问题转化为一维问题，然后参考1中的解法去做。这时我们记录行的上下界（high、low）。此时我们发现在行的上下内即（row∈（low，high)），此时记录每一列的高度h。我们发现只有当h==（high - low）时我们判断有一个高为H的小矩阵。那么怎么处理长度不为1的小矩阵呢？我们不妨将此时的每列高度进行记录，得到数组H，此时我们视为上题的一个变种，即在H数组中寻找连续的全（high-low）子数组。此时可以得到在（low，high）中高为high-low的全部子矩阵个数。那么不难发现我们只需要遍历所有的high，low组合既可以得到我们所想要求的答案。

### 3.560和为k的子数组个数： 

思路：

巧用前缀和。我们统计所有前缀和，在遍历的时候利用    当前前缀和 -之前记录的前缀和   如果结果等于我们想要的k则说明有n个已当前位置结尾的连续子数组和为k

## 旋转数组

### 189.轮转数组

题目：  
给你一个整数数组nums，将数组中的元素右移k个位置

思路：  

思路一： 创建一个新数组  
思路二： 三次反转  
思路三： 这个有点新意为什么我们不一个数字一个数字进行挪移呢？因为我们如果按顺序挪移时，会存在后续数字找不到的情况。但是我们不妨抛弃这该死的顺序，我们直接挪。比如我1挪到3 ，后我直接3挪到5，然后5+2超过6所以挪到1。这时候从1能到达的所有点位遍历完成。然后从2点开始更改直到回到2点。最多从k点开始遍历一定能完成。此外，我们只需要记录替换次数，到达该有的次数就可以截止【就像人生当中，当我们要实现一个完整的目标。我们选定了起点就一定要沿着路尽力走到头，看到这条路无法彻底实现梦想，不妨换个节点开始继续努力，不断尝试，直到实现目标】
